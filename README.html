Multi-peg Tower of Hanoi

We consider Stewart's algorihm for solving the multi-peg Hanoi Tower problem.

Algorithm

Stewart's algorithm solves the p-peg n-disc from source to destination as follows:

  • choose a height h∈{1,…,n−1}h∈{1,…,n−1} (see below) ;
  • move the (n−h)(n−h)-high upper part of the tower to any peg different from source and destination, say number peg ii ;
  • move the hh-high lower part from source to destination using all pegs except ii ;
  • move the (n−h)(n−h) from ii to destination.

(Short) analysis

The number of steps S(n,p)S(n,p) to solve the p-peg n-disc problem satisfies the recurrence relation:

∀p≥3,∀n,S(n,p)=min1≤h≤n−12S(n−h,p)+S(h,p−1),∀p≥3,∀n,S(n,p)=min1≤h≤n−12S(n−h,p)+S(h,p−1),
with S(1,2)=1S(1,2)=1 and S(n,2)=∞S(n,2)=∞ for all n≥2.n≥2.

For any (n,p)(n,p), an optimal choice is any element of the argmin set h(n,p)h(n,p). The number of paths $N(n,p)$ that can generated by the algorithm satisfies the following recurrence:

N(n,p)=∑h∈h(n,p)N(h,p)+N(n−h,p−1),N(n,p)=∑h∈h(n,p)N(h,p)+N(n−h,p−1),
